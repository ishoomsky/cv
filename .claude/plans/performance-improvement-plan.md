## Диагноз: три независимые причины, не одна

1. **Главная — почти весь main-thread work (92–93%) попадает в категорию "Other"**, а не в Script
   Evaluation. Это сигнатура постоянно работающих canvas/WebGL `requestAnimationFrame`-циклов
   (`StarFieldComponent`, `AsteroidsComponent`, `PlanetComponent` three.js-сцена — все три крутятся
   непрерывно), а не тяжёлого JS-кода приложения. Именно это держит TBT и Speed Index высокими даже
   там, где FCP/LCP отличные (desktop: LCP 0.4 сек, но Speed Index 7.2 сек).
2. **Изображения портфолио — 5 018.7 KiB из 7 354 KiB общего трафика (68%)**, при этом решение уже
   существует в кодовой базе, просто не применено везде.
3. **CLS 0.157 на мобильном — на 100% из-за одного decorative-элемента `.scanline`**, который двигает
   layout-свойство вместо composited transform. Это же помечено отдельно как «Избегайте
   некомбинированных анимаций».

- [x] Конвертированы в WebP q80 (sharp), разрешение исходное 1375×768 (близко к 2x DPR контейнера):

      `decoding="async"` на `<img>` для карточек ниже фолда уже открытой модалки. Проверено в
      браузере: на первой загрузке страницы ни один webp не запрашивается; по клику на

## Фаза 2 — CLS: анимация `.scanline` (мобильный CLS 0.157 → цель <0.05)

**Файл:** `src/app/content-container/content-container.component.scss:26-29`

```scss
@keyframes scan {
  0%   { top: -10%; opacity: 0.7; }
  100% { top: 110%; opacity: 0; }
}
```

Lighthouse прямым текстом: элемент `div.scanline` — «Неподдерживаемое свойство CSS: `top`» и он же —
единственный источник всех layout-shift событий в CLS-трассе (0.157 суммарно, по 5 повторяющимся
сдвигам за один 5-секундный цикл анимации `scan 5s linear infinite`).

- [x] `@keyframes scan` переписан на `transform: translateY(-100%) → translateY(calc(100% / 0.12))`;
      `.scanline` запаркован на `top: 0` с `height: 12%` (проценты собственной высоты становятся
      фиксированной долей панели — так transform выражает полный проход в собственных координатах).
      Проверено в браузере: keyframes компилируются в `translateY(-100%) → translateY(833.333%)`,
      анимация идёт на компоситоре (computed transform не обновляется main thread'ом — это и есть
      признак composited), полоса визуально движется.
- [ ] После деплоя перепроверить в PSI, что `.scanline` ушёл из «Избегайте некомбинированных
      анимаций» и мобильный CLS приблизился к desktop-показателю (0.035).

## Фаза 3 — `.online-pulse`: некомпозитная анимация (низкий приоритет)

**Файл:** `content-container.component.scss:31-34` — анимирует `color` и `text-shadow`, тоже помечено
Lighthouse как некомпозитная анимация. Не даёт CLS (не layout-свойства), но заставляет браузer
перерисовывать элемент каждый кадр, пока `#OPENTOWORK` смонтирован (постоянно, весь сеанс).

- [x] Заменено на пульс через `opacity` (.68 ↔ 1): яркое свечение (`text-shadow 0 0 12px`) стало
      статичным состоянием, «дыхание» делает только opacity — компоситный слой вместо покадрового
      repaint. Визуально эффект пульса сохранён.

## Фаза 4 — Постоянные rAF-циклы (canvas/WebGL) — главный вклад в TBT/Speed Index

Main-thread work "Other" ≈30-35 сек — Angular-код (`Script Evaluation`) отвечает лишь за ~2 сек из
этого. Компоненты с `requestAnimationFrame`: `PlanetComponent` (three.js), `StarFieldComponent`,
`AsteroidsComponent`, плюс `NebulaComponent` (по документации рисует статично, но стоит перепроверить,
что он не перерисовывается лишний раз на каждый resize/scroll-событие).

- [x] `visibilitychange` не нужен: браузер сам не вызывает rAF в скрытых вкладках. Вместо этого во
      все троттленные циклы добавлен clamp dt (≤4 кадров), чтобы возврат из фоновой вкладки не
      «телепортировал» объекты.
- [x] Троттлинг rAF: `star-field`, `asteroids`, `planet` — до ~30fps; `nebula` — до ~15fps (её дрейф —
      синус с периодом в минуты, межкадровое движение суб-пиксельное; вопреки CLAUDE.md она рисовала
      full-screen `drawImage` на 60fps). Движение переведено на dt-масштаб (в единицах 60fps-кадра),
      визуальная скорость дрейфа/вращения не изменилась. WebGL-рендер планеты — самая дорогая
      покадровая работа страницы — теперь выполняется вдвое реже.
- [x] `planet`: DPR-клэмп на мобильном вьюпорте (<1200px, в синке с desktop-брейкпоинтом) снижен
      2 → 1.5 (~44% меньше пикселей на кадр). На десктопе остался ≤2; `antialias` не тронут, чтобы не
      менять видимое качество на десктопе. Shadow maps в сцене нет.
- [x] Бонус: у `nebula` не было reduce-motion ветки (в отличие от остальных canvas-слоёв) — добавлен
      статичный кадр по паттерну star-field.
- [x] Визуальная проверка в браузере (desktop 1503px): все слои на месте, планета рендерится, панель и
      портфолио работают.
- [ ] Прогнать глазами мобильные брейкпоинты 375/768/1024px (по паттерну из `ui-redesign-plan.md`)
      после деплоя.
